What is Rate Limiting?

Rate limiting restricts the number of requests a user can make to an API within a specific timeframe. This prevents:

  • Brute Force Attacks: Too many password guesses
  • DDoS Attacks: Overwhelming server with requests
  • Resource Abuse: Using too much server capacity
  • API Abuse: Scraping data or automated attacks

Expected Behavior:

A properly configured rate limiter should:

  1. Allow normal usage (1-2 requests/second)
  2. Warn or slow down moderate abuse (5-10 requests/second)
  3. Block excessive abuse (15+ requests/second)
  4. Return HTTP 429 (Too Many Requests) status code
  5. Reset after the time window expires
